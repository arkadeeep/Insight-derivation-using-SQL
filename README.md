# 🛒 Jensen's Retail SQL Analysis

## 📌 Project Overview
This project focuses on analyzing retail operations data for Jensen's using SQL. The objective is to extract actionable insights related to customer behavior, staff performance, inventory trends, and store efficiency.

The analysis simulates real-world business scenarios where data-driven decisions are critical for optimizing sales, workforce productivity, and product management.

---

## 🎯 Problem Statement
As a Data Analyst at Jensen's, the goal is to:
- Understand customer purchasing patterns
- Evaluate staff sales performance
- Identify top-performing products and categories
- Improve inventory and store-level decision-making

---

## 🗂️ Dataset Description
The dataset consists of multiple relational tables representing:
- Customers
- Orders & Order Items
- Products & Categories
- Stores
- Staff

These tables are linked using primary and foreign keys to enable comprehensive analysis.

---

## ⚙️ Tech Stack
- SQL (Advanced querying, joins, subqueries, window functions)
- RDBMS (MySQL / PostgreSQL / SQL Server – update as applicable)

---

## 🔍 Key Analyses Performed

### 1. Store-Level Sales Analysis
- Total products sold by each store
- Store-wise order distribution

### 2. Product & Inventory Insights
- Cumulative sales trends per product over time
- Products never ordered (inventory inefficiencies)
- Highest priced product per category
- Median price analysis

### 3. Revenue & Category Performance
- Top-selling products by total revenue (quantity × price)
- Top 3 most sold products by volume

### 4. Customer Behavior Analysis
- Highest spending customer
- Customers ordering across all product categories
- Orders per customer per store

### 5. Staff Performance Evaluation
- Staff with no sales
- Staff outperforming average sales
